### 但为什么没有投影效果？

**核心原因：这种 IBL（Image-Based Lighting）+ AO（Ambient Occlusion）的架构，本质上是"光从四面八方同时照来"，不支持方向性投影阴影。**

具体从三个层面解释：

#### 层面 1：环境光照采样方式

在 [pbr/shade.py:L298-L308](file:///d:/localSpace/relighting/RTR-GS/pbr/shade.py#L298-L308)，PBR 的漫反射光照计算为：

```python
# 采样环境光diffuse cubemap — 只在法线方向采样！
diffuse_light = dr.texture(light.diffuse[None, ...], sampling_normals.contiguous(), ...)

diffuse_light = diffuse_light * occlusion                           # AO校正
diffuse_light = diffuse_light + (1.0 - occlusion) * irradiance      # 间接光（=0）
```

关键在于：`light.diffuse` 是一个**预积分漫反射 cubemap**——它已经把整个半球的光照卷积为一个值。每个法线方向对应一个 diffuse 颜色，**不区分光从半球内的哪个具体方向来**。

类比理解：这就像房间天花板装了均匀的柔光灯，而不是一个聚光灯。所以无论物体 A 有没有挡在物体 B 和光源之间，只要 B 的法线朝向亮的半球方向，B 就会被照亮。

#### 层面 2：遮挡体的本质是 Ambient Occlusion

遮挡体烘焙流程（[baking.py](file://d:/localSpace/relighting/RTR-GS/baking.py)）：

```
每体素位置 → 6面cubemap渲染 → 判断每方向是否被几何遮挡 → SH系数投影
渲染时 → 三线性插值SH → GGX采样(roughness=1, 256 rays) → 平均 → 单一标量occlusion值
```

结果：**每个像素得到一个标量值 `[0,1]`**，表示"该点周围半球有多少比例可见"。

这是一个**各向同性的环境光遮蔽**：
- `occlusion ≈ 0.3`：该点在凹陷/拐角处，大部分方向被附近几何遮挡
- `occlusion ≈ 1.0`：该点在开放区域，几乎全方向可见
- 它不区分是"左边被挡住了"还是"右边被挡住了"

因此即使物体 B 相对光源方向被 A 遮挡，只要 B 在其他方向是开放的，occlusion 值仍然接近 1.0。

#### 层面 3：SH 度数太低，无法编码方向性

遮挡体使用 degree=3 的球谐（仅 9 个系数！），只能表示极低频的可见性分布。假设物体 A 在 B 的左侧方向投射了一个尖锐阴影，9 个 SH 系数根本无法区分"左侧有遮挡"和"平均遮挡多一些"。
