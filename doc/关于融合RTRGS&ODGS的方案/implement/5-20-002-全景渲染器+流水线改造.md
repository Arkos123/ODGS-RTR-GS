## 阶段3：等距柱状渲染器

### 新增 [gaussian\_renderer/render\_equict.py](file:///home/huangpengyue/projects/RTR-GS/gaussian_renderer/render_equirect.py) (\~400行)

等距柱状渲染器，使用 **ODGS CUDA 光栅化器**（`odgs_gaussian_rasterization`）替代 RTR-GS 的透视光栅化器。

**核心设计：多通道前向渲染（Multi-Pass Forward Rendering）**

由于 ODGS 光栅化器不支持 `features` 张量机制，采用多通道方案传递属性：

| 通道           | colors\_precomp =                    | 输出               | 用途               |
| ------------ | ------------------------------------ | ---------------- | ---------------- |
| Pass 1       | 前向着色混合后的颜色（PRT + 反射）                 | `rendered_image` | 最终渲染图像           |
| Pass 2       | 法线编码为 `n*0.5+0.5` (范围\[0,1])         | `normal_map`     | 逐像素法线（解码：`*2-1`） |
| Pass 3 (PBR) | 基础颜色 RGB                             | `base_color_map` | PBR 反照率          |
| Pass 4 (PBR) | 打包(R=roughness, G=metallic, B=depth) | `packed_map`     | 粗糙度/金属度/深度       |

**关键技术细节：**

- 各向同性缩放：ODGS 存储 `[N,3]` 各向同性缩放（3 个值相同），传给 ODGS rasterizer 前通过 `scales.mean(dim=-1, keepdim=True).repeat(1, 3)` 显式保证各向同性
- 等距柱状视线方向计算：`_compute_equirect_view_dirs()` 根据像素坐标 `(u,v)` 计算 `lon/lat` → `(dir_x, dir_y, dir_z)` → 用 `c2w` 旋转到世界空间
- PBR 着色使用 `cubemap` 作为光源（类似重光照模式），通过 `pbr_shading()` 函数
- 遮挡查询复用 `gs_ir.recon_occlusion()`（与投影模型无关）
- 纬度加权损失：`est_wsmap()` 生成 `cos(lat)` 权重图，自动启用 `WS-SSIM`

## 使用方式

```bash
# 非PBR等距柱状训练（Stage 1）
python train.py -t render_ref_equirect \
    -s <OpenMVG_dataset_path> \
    -m <output_path>/stage1 \
    --ref_map --compute_with_prt \
    --diffuse_iteration 3000 --skip_eval

# PBR等距柱状训练（Stage 2，需先运行Stage 1和baking）
python train.py -t render_ref_pbr_equirect \
    -s <OpenMVG_dataset_path> \
    -m <output_path>/stage2 \
    -c <output_path>/stage1/checkpoint/chkpnt30000.pth \
    --occlusion_path <output_path>/stage1/checkpoint/occlusion_volumes.pth \
    --ref_map --compute_with_prt --iterations 40000
```
